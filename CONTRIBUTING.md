# 贡献指南

感谢你愿意一起维护 `ai-dev-rules`。

## 提交原则

- 保持规则短、明确、可执行
- 尽量不引入和主题无关的内容
- 不要把具体业务项目名写进通用规则层
- 变更要能说明“为什么要加这条规则”

## 建议的贡献方式

1. 先确认你要改的是通用规则，还是某个技术栈的专用规则。
2. 如果是项目专属内容，优先改 `rules/project-template.md` 或你自己的项目侧规则。
3. 如果是新增通用规则，尽量放到最合适的规则文件里，不要堆进 `core.md`。
4. 如果是 Codex 适配相关内容，放到 `adapters/codex/` 下。

## 推荐提交内容

- 新增一条确实通用的规则
- 改善现有规则的可读性
- 补充一个容易踩坑的场景说明
- 优化目录结构或使用说明

## 不建议提交的内容

- 和主题无关的长篇说明
- 仅对单个项目成立的业务细节
- 过长的示例代码
- 会让规则失去通用性的专有名词

## 变更检查

提交前请先问自己三件事：

1. 这条规则是否真的通用？
2. 这条规则是否能被 AI 稳定执行？
3. 这条规则是否应该放在更具体的子文件里？
