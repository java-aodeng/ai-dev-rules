# 项目路线图

## 目标

让 `ai-dev-rules` 成为一个可以被多种 AI 编程助手直接复用的开发规范仓库。

## 当前重点

- 先稳定通用规则层
- 先稳定 Java 和前端两个常见技术栈规则
- 先让 Codex 适配层能直接读懂同一套规则
- 先把项目模板和开源基础件补完整

## 下一阶段

### v0.1

- 仓库结构完成
- README、LICENSE、CONTRIBUTING、SECURITY、CHANGELOG 到位
- 通用规则、后端规则、前端规则、项目模板拆分完成

### v0.2

- 补充更多技术栈规则模板
- 补充更细的项目覆盖示例
- 优化 Codex 适配层的说明方式

### v1.0

- 形成稳定的通用规则体系
- 能直接复制到新项目使用
- 具备可发布、可 fork、可二次定制的最小完整形态

## 发布标准

发布一个版本前，至少满足下面几条：

1. 核心规则没有明显冲突
2. README 能让新用户快速理解怎么用
3. 目录结构清晰，没有多余文件
4. 规则没有绑定某一个具体业务项目名
5. 适配层不会影响其他 AI 的使用

## 还没定下来的点

- 是否增加更多语言规则
- 是否补充更细的项目模板
- 是否增加一个面向人类的极简使用示例
