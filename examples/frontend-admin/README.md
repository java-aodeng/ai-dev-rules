# 前端管理端接入示例

这是一个示例，展示如何把 `ai-dev-rules` 接入到 Ant Design Pro 管理端项目里。

## 示例目标

让前端在开发时先遵守通用规则，再遵守前端专用规则和项目自己的约定。

## 你需要做什么

### 1. 在项目根目录放一个 `AGENTS.md`

把项目自己的规则写进去，比如：

```md
# AGENTS.md

- 接口与类型一律用 openapi 生成
- 禁止手改生成物
- 请求参数优先复用 API 类型
- 用户触发的接口调用要统一错误提示
- 先读项目结构，再开始改动
```

### 2. 让 AI 先读通用规则

推荐顺序：

1. 读 `ai-dev-rules/rules/core.md`
2. 读 `ai-dev-rules/rules/frontend-ant-design-pro.md`
3. 读项目根目录 `AGENTS.md`
4. 再开始改前端页面或接口调用

### 3. 接入前端工程自己的约定

在具体项目里，通常还会再补这些内容：

- 页面目录结构
- 请求封装方式
- 路由和权限的使用方式
- 表单和表格的通用写法

### 4. 处理 openapi 生成物

前端项目里，如果后端接口变了，先执行 openapi 生成，再改业务代码。

## 最小接入示例

```text
project/
├── AGENTS.md
├── src/
│   ├── pages/
│   └── services/
└── ai-dev-rules/
    ├── rules/
    └── adapters/
```

## 适合什么场景

- 你想让 AI 在前端项目里遵守同一套请求和错误处理规则
- 你想把 openapi 生成和前端约定固定下来
- 你想先看一个前端示例，再复制到自己的仓库
