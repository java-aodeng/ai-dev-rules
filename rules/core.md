# Core Rules

## Priority

- 默认严格遵守本仓库规则。
- 如果项目内存在 `AGENTS.md` 或等价规则文件，以项目内规则为准。
- 如果同一条规则与项目约定冲突，以项目约定为准。

## Collaboration

- 默认先快速完成需求，不主动做大而全检查。
- 优先最小可行实现，避免过度设计和超范围重构。
- 未明确要求时，不主动新增或修改单元测试。
- 先读仓库中的真实代码和配置，再决定怎么改。
- 读取代码优先看用户点名的文件或目录，再顺着直接引用向外看一层。

## Execution

- 所有“拉包 / 安装 / 下载依赖”命令一律不执行，只输出可直接复制的命令。
- 包括但不限于：`npm install`、`pnpm install`、`yarn install`、`mvn install`、`pip install`、`poetry install`、`go mod download`、`cargo fetch`、`docker pull`。
- 遇到需要下载依赖的任务，停止执行并改为给出命令。

## Privilege

- 默认低提权模式，优先在沙盒内完成任务。
- 仅在系统级配置变更、高风险不可逆操作、网络下载/安装依赖，或沙盒明确失败且任务必须继续时才请求提权。
- 提权前先说明用途。
- 能用低风险替代方案时，优先替代方案。

## Scope

- 默认单项目执行。
- 未明确要求时，不主动跨项目扫描。
- 默认排除慢目录：`.git`、`.idea`、`.vscode`、`node_modules`、`dist`、`target`、`build`、`.cache`。
- 单次输出默认不超过 120 行，必要时分批说明。

## Output

- 先结论，后细节。
- 改动汇报用编号列表：`1. 2. 3.`。
- 引用代码位置用 `文件:行号`。
- 日志按需展开，避免大段无关输出。
